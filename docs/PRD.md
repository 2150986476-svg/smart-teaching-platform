**1. 产品概述**

**1.1 产品定位**

AI智能教学平台是一个面向高等院校的智能化教学辅助系统，深度融合Coze AI Agent能力，为教师和学生提供从\"教\"到\"学\"的全链路智能化支持。平台以课程为核心单元，以AI驱动教学内容生成、学习路径推荐、学情智能分析，旨在提升教学效率、实现个性化学习。

**1.2 产品愿景**

让每一位教师拥有AI助教，让每一位学生获得个性化学习伴侣。

**1.3 目标用户**

  -------------- ------------------------------------ ----------------------------------------
  **用户角色**   **描述**                             **核心诉求**
  教师           高校授课教师/讲师/教授               降低备课负担，快速掌握学情，自动化管理
  助教           辅助教师答疑、批改的学生或工作人员   协助答疑，跟踪学习进度
  学生           在校大学生/研究生                    随时提问，个性化练习，看到成长
  教务管理员     院系教务人员                         便捷账号管理，教学数据可视化
  -------------- ------------------------------------ ----------------------------------------

**1.4 核心价值**

-   AI驱动的课程知识库：自动将PDF/Word/PPT/Markdown构建为Coze知识库，精准RAG问答

-   智能助教7×24在线：学生对课程内容提问，AI基于知识库即时作答

-   自适应习题生成：基于课程内容自动生成选择题，辅助自测和巩固

-   全方位学情分析：多维度分析学生能力，生成能力雷达图

-   降低管理成本：Excel批量导入、密码重置、课程管理高度自动化

**2. 业务场景分析**

**2.1 场景一：教师备课与课程搭建**

张老师本学期开设《人工智能导论》课程。他登录平台后创建课程，上传已有的教案PPT、参考书PDF和讲义Word文档。系统自动将这些资料推送到Coze平台构建知识库，并绑定AI助教到课程上。整个课程搭建过程不到10分钟。

**2.2 场景二：学生自主预习与提问**

学生小李登录平台进入《人工智能导论》课程，在课程问答界面输入\"什么是反向传播算法？\"。AI助教基于课程资料（而非通用互联网知识）给出精准回答，并附上参考资料章节。小李可以连续追问，直到完全理解。

**2.3 场景三：学生自我检测**

复习阶段，学生小王点击\"生成习题\"，系统基于课程内容自动生成10道选择题。小王在线作答后，系统自动批改并展示错题解析。错题自动收入错题本，供后续针对性复习。

**2.4 场景四：教师学情掌握**

期中阶段，张老师打开学情分析面板，可查看每位学生的AI提问次数、答题正确率和薄弱知识点分布。能力雷达图直观展示全班学生的掌握情况对比，帮助张老师调整教学重心。

**5. 非功能需求**

**5.1 性能需求**

-   页面首屏加载时间 ≤ 2秒

-   AI问答首次响应 ≤ 5秒（含Coze API调用）

-   习题生成 ≤ 8秒

-   支持 ≥ 500人并发在线

-   单文件上传 ≤ 50MB，批量 ≤ 200MB

**5.2 安全需求**

-   密码采用bcrypt加密存储

-   JWT Token认证，过期后需重新登录

-   API接口防重放攻击，关键接口限流

-   数据隔离：学生只能看到自己所属课程的信息

-   XSS/CSRF防护，输入过滤和输出编码

**5.3 可用性需求**

-   系统可用性 ≥ 99.5%

-   支持自动故障恢复，关键服务高可用部署

-   数据每日自动备份，保留30天

**5.4 兼容性需求**

-   Web端：Chrome 90+、Edge 90+、Safari 14+、Firefox 90+

-   移动端：iOS 14+ / Android 10+

-   支持文件格式：PDF、.docx、.pptx、.md、.txt、.xlsx

**6.2 系统分层架构**

系统采用经典前后端分离架构，整体分为四层：

**展示层：**Web端（React）和移动端（React Native）通过HTTPS与API网关通信。

**接入层：**Nginx反向代理 + Spring Cloud Gateway，负责路由、认证鉴权、限流熔断。

**业务层：**Spring Boot微服务集群，按业务域拆分为用户服务、课程服务、答题服务、学习分析服务等。

**数据层：**PostgreSQL主从 + Redis缓存 + MinIO对象存储。

外部集成：通过Coze OpenAPI深度集成Knowledge API（知识库管理）和Bot API（AI问答）。

**6.3 Coze集成架构**

**知识库自动构建流程：**

教师上传资料 → 后端存储至MinIO → 异步调用Coze Knowledge API创建知识库 → 上传文档 → 轮询解析状态 → 更新数据库状态。

**AI问答流程：**

学生提问 → 调用Coze Bot API（携带知识库ID和上下文）→ RAG检索 → 生成回答 → 存储对话记录 → 返回前端。

**习题生成流程：**

学生生成习题 → 请求Coze Bot（课程内容摘要为输入）→ Coze返回JSON格式题目 → 解析存储并返回前端。

**6.4 Coze API调用规划**

  ------------- ------------------------- -------------- ---------------
  **API类别**   **接口**                  **用途**       **频率**

  Knowledge     Upload Document           上传文档       每次上传文件
  Knowledge     Get Dataset Status        查询解析状态   轮询≤5次/文件
  Bot           Chat                      AI问答         每次学生提问
  Bot           Chat（带system prompt）   生成选择题     每次生成习题
  ------------- ------------------------- -------------- ---------------

**7. 核心数据模型设计**

采用PostgreSQL 16，以下为核心表结构设计。完整DDL将在技术设计阶段输出。

**7.1 核心实体关系**

-   课程（Course）←→ 教师（User）：多对一

-   课程（Course）←→ 学生（User）：多对多（通过选课表）

-   课程（Course）→ 课程资料（Material）：一对多

-   课程（Course）→ Coze知识库（KnowledgeBase）：一对一

-   课程（Course）→ AI助教绑定（AiAssistant）：一对一

-   学生（Student）→ 对话记录（ChatRecord）：一对多

**9. 界面设计指引**

**10. 开发路线图**

-   前后端项目脚手架搭建，CI/CD配置

-   用户认证系统（JWT登录/权限控制）

-   用户管理CRUD

-   数据库初始化

-   开发/测试环境部署

**第二阶段：课程管理与Coze集成（第3-5周）**

-   课程CRUD功能

-   课程资料上传（MinIO存储）

-   Coze Knowledge API集成

-   Coze Bot API集成（AI问答）

-   学生选课与Excel导入

**第三阶段：答题系统与学情分析（第5-7周）**

-   习题自动生成（Coze Prompt）

-   性能优化与安全检测

-   UAT用户验收

-   生产部署上线

**11. 风险识别与应对**

  ------------------ ---------------- ---------- ----------------------------------------
  **风险**           **影响**         **概率**   **应对措施**
  Coze API不稳定     AI问答响应慢     中         前/后端超时重试；熔断降级返回友好提示
  资料格式解析失败   知识库构建失败   中         明确支持格式，上传时格式校验；失败重试
  Coze API限流       高峰时提问受限   低         请求排队/限流；前端排队提示
  数据量大查询慢     学情分析加载慢   低         物化视图/Redis缓存/分表策略
  教师误操作         数据丢失         中         软删除+二次确认+回收站
  ------------------ ---------------- ---------- ----------------------------------------
