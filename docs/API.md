**1. 通用约定**

**1.1 基础URL**

所有API基于以下基础URL：

**http://api.teaching-platform.com/api/v1**

开发环境：http://localhost:8080/api/v1

**1.2 统一响应格式**

所有接口返回统一的JSON结构：

{ \"code\": 200, // 业务状态码 \"message\": \"success\", // 提示信息 \"data\": { \... }, // 响应数据（可能为null） \"timestamp\": 1719984000000 // 服务器时间戳 }

**分页接口额外返回：**

{ \"code\": 200, \"message\": \"success\", \"data\": { \"records\": \[\...\], // 当前页数据列表 \"total\": 100, // 总记录数 \"page\": 1, // 当前页码 \"pageSize\": 20 // 每页大小 }, \"timestamp\": 1719984000000 }

**1.3 认证方式**

使用JWT Bearer Token认证。登录接口返回Token后，后续请求需在HTTP Header中携带：

**Authorization: Bearer \<token\>**

Token过期时间默认24小时（可通过system\_config配置）。过期后需重新登录。

**1.4 HTTP方法语义**

  ---------- ------------------- ----------
  **方法**   **语义**            **幂等**
  GET        查询资源            是
  POST       创建资源/触发操作   否
  PUT        全量更新资源        是
  PATCH      部分更新资源        是
  DELETE     删除资源            是
  ---------- ------------------- ----------

**1.5 通用请求头**

  ----------------- ------------------ ----------------
  **Header**        **说明**           **必填**
  Authorization     Bearer Token       是（除登录）
  Content-Type      application/json   是（POST/PUT）
  Accept-Language   zh-CN / en-US      否
  X-Request-Id      请求追踪ID         否
  ----------------- ------------------ ----------------

**1.6 全局错误码**

  ------------ -------------------- --------------
  **错误码**   **说明**             **HTTP状态**
  200          成功                 200
  400          请求参数错误         400
  401          未认证/Token过期     401
  403          权限不足             403
  404          资源不存在           404
  409          资源冲突（如重复）   409
  422          请求体校验失败       422
  429          请求频率过高         429
  500          服务器内部错误       500
  503          服务暂不可用         503
  ------------ -------------------- --------------

**2. 认证模块**

认证模块负责用户登录、登出和Token刷新。教师使用工号/邮箱登录，学生使用学号登录。

**2.1 教师/管理员登录**

**POST /auth/login**

教师/管理员/助教使用工号（或邮箱）和密码登录，返回JWT Token。登录成功后记录最后登录时间和IP。

**请求体（JSON）：**

  ---------- ---------- ---------- ------------------------------------
  **参数**   **类型**   **必填**   **说明**
  username   string     是         登录名（工号/邮箱）
  password   string     是         登录密码（明文，服务端bcrypt验证）
  remember   boolean    否         记住登录（Token有效期延长至7天）
  ---------- ---------- ---------- ------------------------------------

**成功响应示例：**

{ \"code\": 200, \"data\": { \"token\": \"eyJhbGciOiJIUzI1NiIs\...\", \"expiresIn\": 86400, \"user\": { \"id\": 1, \"username\": \"T2024001\", \"realName\": \"张老师\", \"role\": \"teacher\", \"avatar\": null, \"department\": \"计算机学院\" } } }

**错误码：**

  ------------ ------------------
  **错误码**   **说明**
  400          用户名或密码为空
  401          用户名或密码错误
  403          账号已被禁用
  ------------ ------------------

**2.2 学生登录**

**POST /auth/student/login**

学生使用学号和密码登录。首次登录时返回强密码修改标识（firstLogin=true），前端应强制跳转修改密码页。

**请求体（JSON）：**

  ---------- ---------- ---------- -----------------------------
  **参数**   **类型**   **必填**   **说明**
  username   string     是         学号
  password   string     是         登录密码
  captcha    string     否         验证码（连续3次失败后必填）
  ---------- ---------- ---------- -----------------------------

**成功响应示例：**

{ \"code\": 200, \"data\": { \"token\": \"eyJhbGciOiJIUzI1NiIs\...\", \"expiresIn\": 86400, \"firstLogin\": true, \"user\": { \"id\": 100, \"username\": \"2024001001\", \"realName\": \"李同学\", \"role\": \"student\" } } }

**错误码：**

  ------------ ------------------------
  **错误码**   **说明**
  400          学号或密码为空
  401          学号或密码错误
  403          账号已被禁用
  422          需要修改密码后才能继续
  ------------ ------------------------

**2.3 退出登录**

**POST /auth/logout**

用户主动退出登录，服务端将当前Token加入黑名单（Redis），后续使用该Token的请求将被拒绝。

**请求头：**

  --------------- ---------- ---------- --------------
  **参数**        **类型**   **必填**   **说明**
  Authorization   string     是         Bearer Token
  --------------- ---------- ---------- --------------

**成功响应示例：**

{ \"code\": 200, \"message\": \"退出成功\" }

**错误码：**

  ------------ -------------------
  **错误码**   **说明**
  401          Token无效或已过期
  ------------ -------------------

**2.4 刷新Token**

**POST /auth/refresh**

使用当前未过期的Token换取一个新的Token，旧Token立即失效。适用于Token即将过期时的续期操作。

**请求头：**

  --------------- ---------- ---------- ------------------------
  **参数**        **类型**   **必填**   **说明**
  Authorization   string     是         当前有效的Bearer Token
  --------------- ---------- ---------- ------------------------

**成功响应示例：**

{ \"code\": 200, \"data\": { \"token\": \"eyJhbGciOiJIUzI1NiIs\...\", \"expiresIn\": 86400 } }

**错误码：**

  ------------ ---------------------------------
  **错误码**   **说明**
  401          Token无效或已过期（请重新登录）
  ------------ ---------------------------------

**2.5 获取验证码**

**GET /auth/captcha**

获取图形验证码图片。返回Base64编码的图片和对应的验证码Key（用于提交时验证）。

**查询参数：**

  ---------- ---------- ---------- -------------------
  **参数**   **类型**   **必填**   **说明**
  width      int        否         图片宽度，默认150
  height     int        否         图片高度，默认50
  ---------- ---------- ---------- -------------------

**成功响应示例：**

{ \"code\": 200, \"data\": { \"captchaKey\": \"a1b2c3d4-\...\", \"captchaImage\": \"data:image/png;base64,iVBORw0\...\" } }

**错误码：**

  ------------ ------------------------
  **错误码**   **说明**
  429          获取太频繁，请稍后再试
  ------------ ------------------------

  configs\[\].value   string            是         新配置值
  ------------------- ----------------- ---------- ----------

{ \"code\": 200, \"message\": \"更新成功\" }
